# Tech Patterns

複数プロジェクトで再利用できるコード・設計パターンを保管する。

## 対象

- 特定のプロジェクトに依存しない汎用的な実装パターン
- 繰り返し使う処理のベストプラクティス
- ライブラリ・フレームワークの使い方のコツ

## ファイル命名規則

`{topic}-pattern.md` 例: `powershell-wpf-japanese.md`, `sjis-utf8-conversion.md`

## フロントマター

```yaml
---
author: ai
created: YYYY-MM-DD
type: tech-pattern
project: {由来プロジェクト名}
tags: [ai-memory, tech-pattern]
---
```
